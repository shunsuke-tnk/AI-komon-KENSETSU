## 原体験ストーリー（なぜ建設業特化なのか）

### 入社当時：属人性と非効率の塊

私がその会社に入社した頃、業務はとにかく属人性と非効率の塊だった。

全国トップセールスを何人も生み出している会社。しかし、その求めるハードルの高さゆえに人が定着しない。そして、あらゆる業務がアナログ。何をするにも時間がかかる。

しかし、研修は手厚い。当時全国でトップの売り上げを上げ続けていた上司がマンツーマンで教えてくれる。おかげでどんどん営業成績は伸びた。一方、お客様が増えるごとに雪だるま式に業務は増え続け、気づけば毎日0時を越えていた。

**「このままじゃまずい・・・」**

心からそう思った。

### 0時からの業務改善

だから、当時の僕は0時以降にスイッチを入れ直し、業務効率化のための改善にテコ入れし始めた。

まずは見積もり書式。当時、5種類の見積もりが一発で出てくるという特有のシステムを使っており、これがとにかく時間がかかっていた。これを全てExcelで実現しようとした。

しかし、致命的なことに気づく。

**原価表がない。**

そう、この会社には原価一覧すらなかった。あるのは、FAXでもらった一覧票のみ。これを指で照らし合わせて原価を作っていたのだった。信じられない。

だから、数百ある塗料の原価一覧を作ることにした。カタログから調べて、荷姿や仕様も全て転記した。

自分で言うのもなんだが、本当に頑張ったと思う。

### 見積もりシステムの完成

最終的には、数量だけ入れたら原価表と、6種類の耐久性別見積もり書を必要な塗料缶数を自動計算して出力するような見積もりシステムが完成した。

死ぬほど大変だった。

しかし、**2時間以上かかっていた見積もり作成が10分で終わるようになった。**

他社はこんなに細かく見積もりを作る業者はいない。

**「どこよりも見積もりが丁寧だ」**

といつも言われるようになった。

社の見積もりは全てこれに置き換わった。どんどんと売り上げが伸びた。自分で作ったものが何かの役に立ち、素直に嬉しかった。

もっと改善したい。

### ChatGPTとの出会い

ChatGPTが登場したのはこの頃だった。

ここから、ありとあらゆる業務にAIを使い、改善した。

結果、あんなに大変だった見積もりは今や事務員が作成するようになった。日々、大量にあった入力作業や報告は、決まった時刻に送られてくる簡単なフォームに答えたら終わるようになった。

### AI導入による成果

さらに、

- **営業として**：全国トップセールス獲得。個人年間売り上げ1.8億円を5年間継続
- **ブログ更新**：AI活用により、複数記事でSEO1位記録
- **WEBページ**：会社のWEBページを作り直し
- **チラシ作成**：外注をやめ、全て自分で作る
- **広告の運用と分析**：創業40年にして、**過去最高売り上げ・過去最高集客数**を記録

営業をやりながら、自社のマーケティングの全てを担当。

それなのに結果、当時0時まで毎日残業していた業務が、全て合わせても今では**定時に帰る**ようになった。

### なぜ建設業特化なのか

現場の非効率に苦しんできた。
人がいないから、自分のためにAIを学ぶしかなかった。
新しい技術が出るたびに飛びつき、現場の改善のためにどう使うかを日々研究し続けた。

その繰り返しで今がある。

**だから、建設に特化した活用ができる。**

- AIに詳しいだけじゃない。
- 売り上げの上げ方を知っている。
- 現場レベルで何をすべきかを苦しんできたからこそ知っている。

そして、この業界にいるからこそ、この業界の課題を解決したいと思っている。

**だから、このサービスを作った。**

- 自分自身が建設業（リフォーム営業）で苦しみ、AIで解決してきた当事者
- 「建設業×生成AI」を現場レベルで実践してきた唯一の存在
- 業界特有の課題（見積もり、原価管理、属人化）を身をもって知っている
- AIに詳しいだけでなく、売上の上げ方・現場改善の方法を知っている

### 【最重要】クロージングまでの流れ：棚卸し・適正診断

#### なぜ「棚卸し」が最重要なのか

**AI導入ができない本当の理由**：
- 「何を導入していいかわからない」だけではない
- **「どこを改善していいかわからない」**
- **「何が改善できて、何が改善できないかわからない」**
- **「改善すべきポイント自体がわかっていない」**

→ だから、まず**プロが「どこが改善できるのか」を診断する**ステップが必要

#### フロントエンドセミナーでの布石

セミナー内で以下のメッセージを打ち出す：

> 「なぜ多くの会社がAI導入に失敗するのか？
> それは、何を導入していいかわからないだけじゃない。
> **そもそも、どこを改善すべきかがわかっていない。**
> 何が改善できて、何が改善できないか、見えていない。
> だから、まずは**プロの目で診断する**ことが必要なんです。」

→ これを布石として、無料診断への誘導がスムーズになる

#### 無料AI活用診断（30分）の設計

**位置づけ**：フロントエンドの2段目（セミナー → 無料診断 → 契約）

**所要時間**：約30分（Zoom）

**実施内容**：

| ステップ | 内容 | 目的 |
|----------|------|------|
| ① ヒアリング | 現在の業務フロー、課題、困りごとを聞き出す | 現状把握 |
| ② 棚卸し | 業務を一覧化し、どこにAIが使えるかを整理 | 改善ポイントの可視化 |
| ③ 適正診断 | 各業務について「改善できるか/できないか」を判定 | 期待値の調整 |
| ④ 優先順位付け | 「まずここから」というポイントを明確化 | 行動指針の提示 |
| ⑤ ロードマップ提示 | 「これとこれを改善するなら約◯ヶ月」という絵を見せる | ゴールのイメージ化 |

#### 棚卸しで明確にすること

- **改善できること**：AI活用で効率化・自動化できる業務
- **改善できないこと**：AI導入が難しい/効果が薄い業務
- **優先すべきこと**：インパクトが大きく、すぐに着手できるもの
- **後回しでいいこと**：効果はあるが、優先度は低いもの

#### 開発ロードマップの提示

棚卸し・適正診断の結果をもとに：

```
【例】
Phase 1（1〜2ヶ月目）：見積もり作成の効率化
Phase 2（3〜4ヶ月目）：日報・報告書の自動化
Phase 3（5〜6ヶ月目）：提案資料のテンプレート化
```

→ 「これだったら大体◯ヶ月の開発になりそう」という具体的な絵を見せる
→ 「これとこれとこれが改善できる」と明確に示す
→ **何をすべきかがわかり、サービス購入への判断がしやすくなる**

#### この流れがクロージングに効く理由

1. **不安の解消**：「何をすればいいかわからない」が解消される
2. **期待値調整**：「これはできる/できない」が明確になり、ミスマッチを防ぐ
3. **ゴールの可視化**：具体的なロードマップで「契約後の未来」が見える
4. **信頼の獲得**：プロとしての診断力を無料で体験してもらえる
5. **売り込み感の軽減**：「押し売り」ではなく「診断→提案」の自然な流れ

#### まとめ：クロージングまでの全体フロー

```
【フロントエンド1段目】セミナー
    ↓ 布石：「改善すべきポイントがわかっていない」というメッセージ
    ↓
【フロントエンド2段目】無料AI活用診断（30分）
    ↓ 棚卸し・適正診断
    ↓ 「何が改善できて、何ができないか」を明確化
    ↓
【提案】開発ロードマップの提示
    ↓ 「これとこれを改善するなら◯ヶ月」という絵を提示
    ↓
【クロージング】契約
    ↓ 何をすべきかが明確なので、判断しやすい
    ↓
【サービス開始】顧問契約スタート
```

## 8. 料金体系

### 根本思想：「AI人材を雇う」感覚

> **競合コンサル・開発会社との比較ではなく、「社内でAI人材を雇う場合」との比較**

「AIにめちゃくちゃ詳しい専門家をリモートで雇用している感覚」
→ 月10万円でAI人材が手に入るなら圧倒的に安い
→ 「この人に頼んでおけばAI周りはなんとかなる」という安心感

### サービス構成（2プラン構成）

大きく2つのサービスを提供：

|--------|-------------|------|

| **② 自走プラン** | 「内製化チームの立ち上げまで支援」 | **月額50万円**（モニター価格30万円） |

### ① 伴走プラン（月額20万円 → モニター価格10万円）
**「AI専門家があなたの会社に」**

**コンセプト**：チャット相談 + 開発で、AI活用を伴走支援

**サービス内容**：

| 項目 | 内容 |
|------|------|
| **チャット相談** | 無制限。業務課題へのAI活用アドバイス、ツール選定など |
| **開発サポート** | 月1つ程度のAIエージェント・簡易アプリの開発 |
| **環境構築支援** | 社内AI環境の構築に向けたアドバイス・伴走 |

**こんな会社向け**：
- まずはAI活用を始めたい
- 何から手をつけていいかわからない
- 相談しながら少しずつ業務改善したい

**Zoomについて**：
- 資料上は明記しない（スケーラビリティの観点）
- 必要に応じて個別対応（〜10社程度までは柔軟に対応可能）
- 100社規模を想定すると対応が難しくなるため、基本はチャット中心

### ② 自走プラン（月額50万円 → モニター価格30万円）
**「内製化チームの立ち上げまで支援」**

**コンセプト**：伴走プラン + 社内AIチームの立ち上げ支援

**サービス内容**：

| 項目 | 内容 |
|------|------|
| **伴走プランの全内容** | チャット相談無制限 + 月1開発 + 環境構築支援 |
| **キックオフミーティング** | 初回に実施。改善方針・ロードマップを策定 |
| **社員教育** | AIチーム立ち上げのための研修・指導（4名まで参加可能） |
| **チャットフォロー** | キックオフ後も継続的にチャットでサポート |
| **内製化ゴール** | 社内でAI活用を自走できる体制構築 |

**こんな会社向け**：
- 本格的にAIを社内に根付かせたい
- 社員にAIスキルを身につけさせたい
- 外部依存せず、自社でAI活用できる体制を作りたい

**販売動線**：
```
【無料AI活用診断】
    ↓
【ロードマップ提示】
    ↓ ここで「自走プラン：50万円」を提示
    ↓
【クロージング】
    ↓ 検討段階で「モニター価格：30万円」を提示
    ↓
【契約】
```

**ポイント**：
- 最初に正規価格（50万円）を提示し、価値を伝える
- クロージング時に「モニター価格として30万円」に落とすことで、お得感を演出
- 「今だけ」「先着◯社」などの限定感も活用可能

- **イメージ**：週1回相談できるAI顧問
- **こんな会社向け**：まずはAI活用を始めたい、相談相手が欲しい

#### 松プラン（30万円/月）- 上位プラン
- **チャット無制限**：いつでも相談可能
- **Zoom**：月4回×1時間（計4時間）
- **社員研修**：月1回あり
- **開発工数**：月◯時間分込み（検討中）
- **イメージ**：専属のAI担当者が社内に入り込む
- **こんな会社向け**：本格的にAI導入を進めたい、開発も含めて任せたい

### 開発について

- **基本方針**：開発は別料金
- **時間単価**：1.5万円/時間
- **見積もり**：案件ごとに見積もり提示
- **松プランに開発工数を含めるか**：検討中（含めることで「専属AI担当者」の価値が上がり、解約リスクも下がる可能性）

### 契約条件

- **最低契約期間**：3ヶ月
- **試用期間**：最初の1ヶ月は「試用期間」として判断可能
  - 「試用期間」という表現で、雇用との比較イメージを刷り込む
  - 3ヶ月契約のハードルを下げる効果
- **返金保証**：なし

### 価格設定の根拠

1. **AI人材採用との比較**
   - 正社員採用：年間600〜1,000万円以上
   - 本サービス（竹）：年間120万円 → **採用の1/5〜1/8**

2. **部長級・現場責任者レベルで即決可能なライン**
   - 月10万円は決裁しやすい金額帯
   - 高すぎず、安すぎず（安すぎると価値を疑われる）

3. **「業界特化の専門家」としての価値**
   - 汎用AIコンサルより価値が明確
   - 建設業の現場がわかる専門家は希少

### セッション2（2026年1月29日）
**検討した項目：**
1. 建設業向けデモアプリ・ツール構想の整理
2. 料金体系・プラン設計（松竹梅の検討）
3. 価格設定の根本思想の明確化

**料金設計の議論経緯：**

1. **松竹梅の初期案**
   - 梅（5.5万円）：チャットのみ、Zoomなし
   - 竹（10万円）：チャット + Zoom月2時間
   - 松（30万円）：チャット + Zoom月4時間 + 社員研修

2. **レビューで出た懸念点**
   - 梅の存在意義が薄い（5.5万円でチャットだけは割高感）
   - 松の魅力が弱い（Zoomが多いだけでは20万円の差額に見合わない）
   - 開発別料金の「結局いくらかかるかわからない」不安

3. **根本思想の確認**
   > 「競合コンサル・開発会社との比較ではなく、AI人材を雇う場合との比較」
   - AIにめちゃくちゃ詳しい専門家をリモートで雇用している感覚
   - 月10万円でAI人材が手に入るなら圧倒的に安い
   - 「この人に頼んでおけばAI周りはなんとかなる」という安心感

4. **方向性の決定**
   - 梅プラン廃止 → **竹・松の二択構成**に
   - 竹（10万円）：パートタイムのAI顧問
   - 松（30万円）：専属のAI担当者（開発工数込みを検討中）

**決定事項：**
- 竹（10万円）をメインプランとして設計
- 梅プランは廃止方向（割高感があるため）
- 開発は基本的に別料金（時間単価1.5万円）
- 契約期間は最低3ヶ月
- 返金保証は保留

**継続検討事項：**
- 松プランに開発工数（月◯時間）を含めるか
- 含める場合、何時間が適切か
- 返金保証の有無・条件
